Elementos de POO

Comenzando con la teoría: ya has aprendido el punto de partida para la POO, y es el modelado de objetos, sus propiedades y funciones, pero la POO tiene sus reglas y su manera de hacer las cosas para conseguir la eficiencia. Por ello debemos introducirnos en la parte más técnica de este paradigma, la teoría, para darnos cuenta de cómo trabaja la POO.


Como se vio en las ventajas que ofrece POO, esta nos permite realizar varias cosas, pero se debe a las Elementos fundamentales del paradigma, cosas que lo hacen único, y son las mismas por las cuales se ha vuelto tan popular.


Para poder programar en un lenguaje de POO hay que tener claro una serie de conceptos fundamentales como son Objeto, Clase, Herencia, Método, Evento, Atributos y propiedades. Estos conceptos amplían y superan conceptos ya conocidos de la programación tradicional.


Clases

Una clase es una plantilla. Define de manera genérica cómo van a ser los objetos de un determinado tipo. Esta es una definición bastante simple pero a su vez muy acertada, pues una clase hace referencia exactamente a eso, una plantilla, un molde, algo con características predeterminadas para ser utilizado y así crear algo más a partir de ese molde.


Retomemos el ejemplo anterior, la espada, antes de existir una espada debe de haber ese molde que nos facilite la creación de la misma, pues pensar en atributos para cada espada es repetitivo e innecesario si estos suelen ser parecidos. 


                            

                                
 class Espada {
                                    
    constructor (tamaño, material) {
                                      
      this.tamaño = tamaño+"m";
                                      
      this.material = material;
                                      
      this.durabilidad = 100;
                                      
      }
                                      
  };

Entonces como ves, al tener clases estas nos sirve como una plantilla de la cual se pueden derivar más de un objeto, cada vez que realizamos este proceso se conoce instanciar un objeto. De esta manera, con instancias, derivamos objetos de una clase y estos obtienen características de manera hereditaria, como padre e hijo, lo que nos lleva al siguiente punto.


Objetos

Un objeto es una entidad que tiene un determinado estado, comportamiento e identidad, los objetos representan o son cosas. Esta es una definición de objetos, aunque acertada, poco intuitiva, he aquí una más: Un objeto es una colección de datos relacionados y/o funcionalidad (que generalmente consta de algunas variables y funciones, que se denominan propiedades y métodos cuando están dentro de objetos).


Si tomamos el ejemplo que realizamos anteriormente, la clase (plantilla), podremos instanciar un objeto, o en este caso una espada, para ello es necesario introducir los parámetros en la plantilla, veamos: 


                            

                                
  const espadaPiedra = new Espada(0.5, “Piedra”));

                                    
    console.log(espadaPiedra);
                                    

                                    
    console output:
                                    
        espadaPiedra
                                    
            tamaño: "0.5m",
                                    
            material: "Piedra",
                                    
            durabilidad: 100;

Vemos que en respuesta obtenemos por consola el código mostrado, lo que significa que hemos instanciado correctamente nuestra espada de piedra, de igual forma podemos crear más de un objeto tan solo teniendo nuestra clase a disposición. 


                            

                                
  const espadaDiamante = new Espada(0.6, “Diamante”));

                                    
    console.log(espadaDiamante);
                                    

                                    
    console output:
                                    
        espadaDiamante
                                    
            tamaño: "0.6m",
                                    
            material: "Diamante",
                                    
            durabilidad: 100;

Atributos

En POO cada objeto dispone de una serie de atributos que definen sus características individuales y le permiten diferenciarse de otros (apariencia, estado, etc).


Como se ha expuesto con anterioridad, los objetos están compuestos por varios elementos, entre ellos, atributos, y estos son datos de los que se vale el objeto para funcionar o incluso simplemente para existir. Estos representas las características o propiedades que puede tener un objeto y pueden ser prácticamente cualquier cosa: en nuestro objeto espada tenemos una cadena de texto y un número. Veamos más ejemplos de atributos que podría tener espada: 


                            

                                
 class bigEspada{
                                      
    constructor (tamaño, material1,material2){
                                      
      this.color = "#00ff99";
                                      
      this.attackPoints = 80 +" Attacks points";
                                      
      this.tamaño = tamaño+"m";
                                      
      this.material = [material1, material2];
                                      
      this.durabilidad = 100;
                                      
   }
                                      
 }

Y así sin más, hemos creado otra plantilla para hacer espadas más grandes, con más atributos, de diferentes tipos de datos incluso hasta con dos materiales en conjunto. Veamos ahora la bigEspada: 


                            

                                
 const espadaSamurai = new bigEspada(1, "Acero", "Oro", 80);
                                      

                                      
      console.log(espadaSamurai);
                                      

                                      
  console output:
                                      
      bigEspada
                                      
          color: "#00ff99"
                                      
          attackPoints: "80 Attacks points"
                                      
          tamaño: "1m"
                                      
          material: Array(2)
                                      
          0: "Acero"
                                      
          1: "Oro"
                                      
          length: 2

Métodos

Es una subrutina que puede pertenecer a una clase u objeto, y son una serie de sentencias para llevar a cabo una acción.


Un método, en palabras sencillas, son acciones, verbos por decirlo de manera literaria. Estas acciones van relacionadas al objeto que pertenecen y estas se ejecutan según las propiedades o atributos del objeto en cuestión.


Para nuestra espada, una acción evidente, seria atacar o cortar, algo que toda arma dentro de un juego debería poder realizar, entonces tratemos de representar esta acción en nuestras espadas: 


                            

                                
class Espada {
                                      
  constructor (tamaño, material, attackPoints){
                                      
      this.tamaño = tamaño;
                                      
      this.attackPoints = attackPoints+" Attacks points";
                                      
      this.material = material;
                                      
      this.durabilidad = 100;
                                      
  }
                                      

                                      
  get atack(){
                                      
      return this.calAtack()
                                      
  }
                                      

                                      
  calAtack(){
                                      
      return this.attackPoints;
                                      
  }
                                      
}

Una vez que nuestras espadas son dotadas de los métodos necesarios para atacar y calcular el daño del ataque solo resta ejecutar el método para atacar a nuestro enemigo. 


                            

                                
console.log(espadaPiedra.atack);
                                      

                                      
  console output:
                                      
      61 Attacks points

Listo! por consola hemos recibido el daño de la espada (atackPoints). Ahora somos capaces de crear las acciones que realizan nuestros objetos.


Eventos

Cuando se habla de eventos nos referimos a la interacción entre objetos, se crean métodos que reaccionan ante ciertas circunstancias (eventos) y estos métodos a su vez se activan y son ejecutados. Entonces un evento en POO seria esa interconexión entre objetos cuando ciertas circunstancias ocurren. Veamos esto con un ejemplo: 


                            

                                
class enemy{
                                      
  constructor (vida){
                                      
      this.vida = vida;
                                      
  }
                                      
  
                                      
  handleAtack(atack){
                                      
      return this.vida = this.vida - atack;
                                      
  }
                                      
}
                                      

                                      
const skeletor = new enemy(300);

Para interactuar con nuestra espada, creamos un nuevo objeto (enemy) un enemigo al cual atacar con nuestro nuevo método (atack). Después de ser creado lo dotamos de un nombre para instanciarlo “skeletor” y le asignamos una cantidad para su vida. 


                            

                                
  let atack = espadaPiedra.atack;
                                      

                                      
  console.log("Atacar a skeletor*: "+espadaPiedra.atack+"!");
                                      
  console.log("Daño critico, vida restante: "+skeletor.handleAtack(atack));
                                      

                                      
  console.log("Atacar a skeleto*: "+espadaPiedra.atack+"!");
                                      
  console.log("Daño critico, vida restante: "+skeletor.handleAtack(atack));
                                      
  
                                      
  console output:
                                      
      Atacar a skeleto*: 61!
                                      
      Daño critico, vida restante: 239
                                      

                                      
      Atacar a skeleto*: 61!
                                      
      Daño critico, vida restante: 178

Por último, asignamos el valor del ataque de la espada a una variable para fácil acceso al dato, y mostramos por consola la interacción entre el ataque de la espada y el daño recibido por la vida del enemigo. Y listo! nuevamente hemos superado otra etapa de la POO, sabiendo estos elementos podemos crear objetos sencillos y agregar la funcionalidad, a partir de este código puedes empezar a practicar, agregando más métodos, condicionales y más, un paso a seguir para agregar más funcionalidad seria condicionar el ataque de la espada para hacer daño según su durabilidad y hacer que la vida del enemigo acabe al llegar a 0 y ya no sea un objeto atacable, en tal caso, debemos hacerlo saber. Pero la POO puede ir más allá, como se observara en sus fundamentos.
